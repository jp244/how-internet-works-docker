IP & DNS

All unique devices on the internet has unique addresses. This address is just a number (similar to a phone number or street address) that is unique to each device at the edge of the network.

Addressing system is a part of one of the most important communication protocols, Internet Protocol (IP). A computer’s address on a network is called its IP Address. A computer sends a message to another using the destination device’s IP while providing the sender’s (origin) device's IP, this way the recipient knows exactly where to send a message back to.

An IP Address is just numbers organized in a hierarchy similar to a home address (street, city, state, county, etc.).
93.184.216.34
8 bits for each part of the address, and traditionally represented country, region, subnetwork, and device.

This version of addressing is called IPv4 and provides more than 4 billion unique addresses for devices connected to the internet. This was designed in 1973 adopted in the early 1980’s, so it hasn’t been able to scale well with the large popularity of the internet today. We’re now in a multi-year transition phase to a new version of IP addressing called IPv6, which uses 128 bits for addresses. This version allows for 340 undecillion unique addresses (more than enough for every grain of sand on Earth to have its own address, and then some).

We usually don’t see IP Addresses when dealing with other computers, since they’re hard for humans to remember multiple patterns of numbers. A system called DNS was designed to map more human-friendly names with IP Addresses. You could think of DNS as a phonebook where computers look up a domain name and could then be returned an IP Address.

Computer uses the domain name to look up an associated IP address which is then used to connect your computer with the destination on the internet. Computers will have to reach out to a DNS server to translate the address, and a DNS server will reply with the proper IP address to navigate to. Due to the nature of the internet being distributed, DNS systems should be as well. All the advantages of having a distributed system still apply, and our DNS servers should be connected in a distributed hierarchy. This splits up responsibility for each top level domain, such as .org, .com, .edu.
